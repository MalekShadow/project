Data Wrangling Report

Internal Use Only — Do Not Distribute

The dataset was sourced from a collection of dog reviews and ratings shared on Twitter. It included tweet text, timestamps, usernames, hashtags, dog names, breed mentions, and assigned ratings. Pandas was used for data structuring and cleaning, while NumPy supported numerical operations and transformations. These tools were selected for their effectiveness in handling social media data.

During the initial inspection, several issues were identified. These included missing values in dog name and breed fields, duplicate tweets, inconsistent date formats, and non-standard rating formats. Some tweets included multiple ratings or used unconventional formats, making parsing challenging. Additionally, text fields contained a mix of emojis, symbols, and irregular spacing.

The wrangling process started by loading the tweets into a Pandas DataFrame. Duplicate entries were dropped to ensure each tweet was unique. Missing values in dog names were partially recovered using pattern recognition within the tweet text. Rating values were extracted using regular expressions and standardized into a numeric format. Date fields were parsed and converted into a consistent datetime format for better filtering and grouping.

NumPy was used to apply transformations on extracted ratings, such as normalizing scales and identifying outliers. Additional features were engineered, including tweet length, presence of hashtags, and detection of media attachments.

By the end of the wrangling process, the dataset was cleaned, consistently formatted, and structured for further analysis. All key fields were validated, text was normalized, and missing or irregular values were addressed. The dataset is now ready for sentiment analysis, rating trend exploration, or machine learning applications.

This report summarizes the steps taken to prepare the dog review and rating tweets for internal research and analysis, ensuring the data is accurate, usable, and well-documented.
